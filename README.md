## 1.5 Expanded Project Idea

In 1–2 paragraphs, explain:

- what your project is,
- what kind of experience it creates,
- what technologies are involved.

**Response:**  
`This project is a high-performance Embedded Logic Analyzer designed to bridge the gap between low-cost microcontrollers and professional-grade debugging equipment. It utilizes a Spartan-7 FPGA as a dedicated high-speed acquisition engine that monitors digital signals generated by an Arduino Nano. By capturing and buffering these signals at hardware speeds, the system identifies specific trigger events and then transmits the resulting waveform data to a monitor via a custom-built UART serial interface. This setup allows for precise, real-time visualization of digital logic without the overhead of expensive, bulky test equipment.

The experience is centered on standalone visibility and portability. As a developer, you gain the ability to "see" exactly what your Arduino code is doing at the hardware level—detecting glitches, timing errors, or logic bugs that a standard IDE serial monitor would miss. By replacing a complex HDMI video pipeline with a streamlined UART-to-Serial Plotter stream, the project creates a plug-and-play debugging environment compatible with any laptop. The technology stack combines Verilog HDL for the FPGA's internal logic (including BRAM-based circular buffers and trigger FSMs), Arduino/C++ for the signal source, and UART communication protocols for data visualization.`

# Technical Implementation Notes

## FPGA Logic Analyzer With Python GUI

This repository includes an 8-bit FPGA logic analyzer implementation for the Real Digital Boolean Board using the Xilinx Spartan-7 `XC7S50-CSGA324-1`. The FPGA captures digital input signals, stores samples in block RAM, and streams the captured data to a Python GUI over the board USB-UART connection.

## Hardware Used

- Real Digital Boolean Board
- Xilinx Spartan-7 `XC7S50-CSGA324-1` FPGA
- 100 MHz onboard clock
- USB cable connected to the board `PROG UART` port
- Pmod/expansion input pins for 8 logic analyzer channels
- Optional Arduino Nano waveform test source

The Boolean Board FPGA inputs are 3.3 V only. Do not connect 5 V signals directly; use a level shifter or voltage divider when testing with a 5 V Arduino Nano.

## Analyzer Input Mapping

| Analyzer channel | Verilog signal | FPGA pin in XDC |
|---:|---|---|
| Ch0 | `pmod_in[0]` | T6 |
| Ch1 | `pmod_in[1]` | T5 |
| Ch2 | `pmod_in[2]` | R5 |
| Ch3 | `pmod_in[3]` | T4 |
| Ch4 | `pmod_in[4]` | R7 |
| Ch5 | `pmod_in[5]` | R6 |
| Ch6 | `pmod_in[6]` | P6 |
| Ch7 | `pmod_in[7]` | P5 |

## Source Files

| Path | Purpose |
|---|---|

## Build and Run

Run a Vivado synthesis check:

```bash
vivado -mode batch -source check_verilog.tcl
```

Install Python dependencies:

```bash
pip install -r requirements.txt
```

Start the GUI:

```bash
python main.py
```

Find the Boolean Board serial port:

```bash
python3 -m serial.tools.list_ports -v
```

On Linux the board may appear as `/dev/ttyUSB0`. On Windows it may appear as `COM3`, `COM4`, or similar. Look for `FTDI` in the detailed port list.
